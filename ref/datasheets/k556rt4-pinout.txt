К556РТ4 / 82S126-class 256x4 open-collector bipolar PROM — Juku D2, D6
======================================================================

Primary artifact: 82s126-556rt4-256x4-oc-prom.pdf
  Signetics 82S126 "1024-Bit Bipolar PROM (256x4)". К556РТ4 is the Soviet
  equivalent of the Intel 3601 / Signetics 82S126 / TI 74S387 family.
  Used on the Juku for D2 (`.037` WAIT/READY) and D6 (`.038` memory-map decode).
Package: 16 pins.

Output type and polarity (the key fact for the dumps)
-----------------------------------------------------
Outputs are OPEN-COLLECTOR and NON-INVERTING. Per the datasheet: a virgin device
is supplied with all outputs at logic "0"; programming fuses a Ni-Cr link to set
a bit to logic "1", and the verify step confirms a programmed output "remains in
the 1 state". So:

  stored/programmed bit = 1  ->  output transistor OFF  ->  with a pull-up, reads HIGH (1)
  virgin bit            = 0  ->  output transistor ON   ->  reads LOW (0)

Therefore an electrically sound reader that supplies a pull-up and records the pin VOLTAGE LEVEL
(HIGH=1) captures the STORED DATA DIRECTLY — the pull-up does NOT invert. The
raw table directly. The original `.038`/`.037` captures used external 1k..4.7k
pull-ups and `raw` = pin level. The 2026-07-19 revision-3 reader separately
checked electrical release, reproduced D2, and proved that the old D6 artifact
had its four data channels reversed. D8 (К155РЕ3) booting from raw `.039` independently excludes a universal
raw/asserted convention error.

Consequence: a uniform logical complement is not justified. The corrected D6
table uses the continuity-confirmed physical channel order and drives all four
HDL outputs directly.

Two chip-enable inputs are active-LOW. Reader revision 3 drives both pins so the
disabled output-release state can be checked independently for each input.

Pinout as wired by tools/rt4_dumper/rt4_dumper.ino (DIP-16)
----------------------------------------------------------
  A0=5  A1=6  A2=7  A3=4  A4=3  A5=2  A6=1  A7=15   (address, 256 words)
  D0=12 D1=11 D2=10 D3=9                            (data out, open-collector)
  CE (2x, active-low) = pin 13 -> GND; pin 14 -> Nano A1
  pin 8  = GND (VSS)
  pin 16 = +5 V (VCC)
Each data output needs a 1k..4.7k pull-up to +5 V. D0-D3 connect to Nano
D10,D11,D12,A0; Nano D13 is deliberately unused because of its on-board LED.

The HDL model (hdl/devices.v decode_prom / wait_prom_037) uses this exact pin
order: address a[0..7] = pins 5,6,7,4,3,2,1,15; raw output bits 0..3 =
D0/pin12 .. D3/pin9. Reader wiring and model indexing agree — no permutation.
