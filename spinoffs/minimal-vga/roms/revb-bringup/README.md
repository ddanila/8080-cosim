# rev B minimum-tier bring-up ROM

`revb_bringup.bin` (16 KiB, org 0x0000) is the **B1 minimum-tier** firmware:
CPU + Memory + I/O(UART) only, no video, no keyboard, no interrupts. It exists so
the minimum tier has something observable to run before ekta37 (which needs video)
can be booted at B2.

Generated by `build_bringup.py` (self-contained 8080 assembler — no toolchain).
Rebuild: `python3 build_bringup.py`. The image is **8080-subset only**, so cosim
is its oracle.

Behavior (all output on the 8251 USART, port 0x08 data / 0x09 ctl-status):

```
VJUGA rev B bring-up
RAM PASS            (walking A5/5A + address-in-cell over 0x4000..0xD6FF)
ROM OK              (whole-ROM checksum == 0 mod 256)
READY               (then the serial monitor)
```

Serial monitor (single-letter commands, hex args): `A`hhhh set address,
`D`ump 16 bytes, `W`hh deposit byte + advance, `G`o (jump to address). The
monitor's interactive path is exercised on the bench (T1.11) and in the twin's
RX-driven test (T1.2); cosim (no serial input) reaches only the quiet wait loop,
so the T1.1 oracle covers init + banner + RAM test + checksum + the TX path.

**cosim coincidence (D1.2):** the ROM polls TxRDY as status-port bit0. cosim has
no USART model and returns the last value written to port 0x09; the 8251 command
word we write has TxEN = bit0 = 1, so the poll reads "ready". This is why the byte
*stream* (not timing) is the oracle. On real hardware / the twin's usart_8251 the
same bit is the genuine TxRDY.

Verified by `spinoffs/minimal-vga/sim/revb_bringup_check.sh` (T1.2).
