## Reversed-card keying (D1.4 / D1.32)

- Correct ext row X = 14.45; reversed (flip about x=50) → X = 85.55 (no socket there).
- Base row centred at x=50.0 (symmetric — maps onto its own socket under the flip).
- **Finding:** because the base connector is centred and symmetric, a reversed card's base pins still enter the base socket; keying can only come from the ext row. With generic 0.1" headers modelled here we cannot prove the ext male pins bottom out on the bare backplane before the base seats, so **mechanical keying is not guaranteed to self-enforce**.
- **Decision (D1.32b):** adopt RC2014-precedent **convention-only** orientation — silk pin-1 arrows on both card and backplane, and a polarity notch mark — and record the residual mis-insertion risk. A generator-emitted blocking post (D1.32a) remains available if bench experience shows convention is insufficient; it needs a card clearance hole at the mirrored-ext X and is deferred to avoid over-design.
